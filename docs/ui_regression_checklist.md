# UI回帰チェックリスト

## 目的
レイアウト修正時に、対象外の項目が改悪されることを防ぐ。

## 運用ルール
1. 変更前に対象を固定する  
   変更する列・画面・セレクタを明記する。

2. 変更は最小差分に限定する  
   目的外のCSS/JSは触らない。

3. 変更後に `git diff` を確認する  
   対象外ファイル・対象外セレクタの差分がないことを確認する。

4. 回帰確認を実施する  
   少なくとも以下を毎回確認する。
   - 親タスク一覧: 完了 / タスク / タイトル / 担当者 / 登録者 / 備考
   - ルーチン一覧: 完了 / 期日 / 担当者 / 登録者 / 備考
   - セル寄せ: 左寄せ・中央寄せが崩れていない
   - 列幅: 指定列のみ変更され、他列は維持されている

5. キャッシュを除外して確認する  
   `Ctrl + F5` で再読み込みし、表示確認する。

6. 問題があれば即ロールバックする  
   1コミット単位で戻せるように小さくコミットする。
